16// Save/resume routines:
17//
18// Engines which aren't resumable can simply ignore all this.
19//
20// Before calling the (per_image,calctype) routines (engine), calcfract sets:
21// "g_resuming" to false if new image, true if resuming a partially done image
22// "g_calc_status" to IN_PROGRESS
23// If an engine is interrupted and wants to be able to resume it must:
24// store whatever status info it needs to be able to resume later
25// set g_calc_status to RESUMABLE and return
26// If subsequently called with resuming true, the engine must restore status
27// info and continue from where it left off.
28//
29// Since the info required for resume can get rather large for some types,
30// it is not stored directly in save_info. Instead, memory is dynamically
31// allocated as required, and stored in image files as a separate block.
32// To save info for later resume, an engine routine can use:
33// alloc_resume(max_size, version)
34// Max_size must be >= max bytes subsequently saved + 2; over-allocation
35// is harmless except for possibility of insufficient mem available;
36// undersized is not checked and probably causes serious misbehaviour.
37// Version is an arbitrary number so that subsequent revisions of the
38// engine can be made backward compatible.
39// Alloc_resume sets g_calc_status to RESUMABLE if it succeeds;
40// to NON_RESUMABLE if it cannot allocate memory
41// (and issues warning to user).
42// put_resume({bytes, &argument}, ..., 0)
43// Can be called as often as required to store the info.
44// Is not protected against calls which use more space than allocated.
45// To reload info when resuming, use:
46// start_resume()
47// initializes and returns version number
48// get_resume({bytes, &argument}, ..., 0)
49// inverse of store_resume
50// end_resume()
51// optional, frees the memory area sooner than would happen otherwise
52//
53// Example, save info:
54// alloc_resume(sizeof(params) + 100, 2);
55// put_resume(sizeof(row), &row, sizeof(col), &col,
56// sizeof(params), params, 0);
57// restore info:
58// version = start_resume();
59// get_resume(sizeof(row), &row, sizeof(col), &col, 0);
60// if (version >= 2)
61// get_resume(sizeof(params), params, 0);
62// end_resume();
63//
64// Engines which allocate a large memory chunk of their own might
65// directly set resume_info, resume_len, g_calc_status to avoid doubling
66// transient memory needs by using these routines.
67//
68// standard_fractal, calcmand, solid_guess, and bound_trace_main are a related
69// set of engines for escape-time fractals. They use a common worklist
70// structure for save/resume. Fractals using these must specify calcmand
71// or standard_fractal as the engine in FractalSpecific.
72// Other engines don't get btm nor ssg, don't get off-axis symmetry nor
73// panning (the worklist stuff), and are on their own for save/resume.
74//
